# Структура файлов алгоритма

### `run_v2.py`
Точка запуска для одной пары изображений `mask/live`.

Что делает:
- читает пути к изображениям из аргументов командной строки;
- загружает изображения и приводит их к диапазону `0..255`;
- создает рабочую конфигурацию алгоритма;
- запускает `run_pipeline`;
- сохраняет результат через `save_result`;
- печатает краткие метрики в терминал.

### `run_ablation.py`
Скрипт для сравнения нескольких вариантов алгоритма.

Что делает:
- запускает алгоритм на подготовленных парах изображений;
- включает/выключает отдельные блоки: сосудистую маску, haze correction, intensity correction, non-rigid preference;
- сохраняет результаты каждого варианта;
- формирует общий `summary.csv`;
- создает визуальную таблицу сравнения.

## Пакет `dsa_v2`

### `dsa_v2/config.py`
Хранит все параметры алгоритма.

Что задает:
- размеры шаблонов и радиус поиска;
- критерии выбора контрольных точек;
- параметры сосудистых масок;
- параметры matching;
- правила выбора `rigid/affine/nonrigid`;
- параметры защиты сосудов;
- параметры яркостной коррекции и подавления фоновой дымки.

### `dsa_v2/io.py`
Работа с изображениями.

Что делает:
- загружает grayscale-изображения;
- нормализует входные кадры;
- переводит численные DSA-изображения в `uint8`;
- строит окно отображения для DSA;
- применяет визуальное улучшение результата;
- сохраняет изображения.

### `dsa_v2/masks.py`
Построение ROI и масок.

Что делает:
- выделяет область анализа `roi`;
- строит узкую сосудистую маску `vessel_mask`;
- строит расширенную `vessel_support_mask`;
- оценивает сосудистое русло `vascular_bed`;
- строит безопасную область регистрации;
- строит карту весов регистрации.

Роль в алгоритме:
- сосуды исключаются из оценки движения;
- `vessel_support_mask` защищает сосудистую область от яркостной коррекции и помогает не брать ложные точки.

### `dsa_v2/features.py`
Выбор контрольных точек.

Что делает:
- считает градиенты и Harris response;
- оценивает информативность ткани;
- выбирает контрольные точки на стабильном фоне;
- избегает сосудов и слабых областей;
- распределяет точки по ROI, чтобы поле движения не строилось по одной локальной зоне.

### `dsa_v2/matching.py`
Локальная оценка смещений контрольных точек.

Что делает:
- сравнивает патчи `mask` и `live`;
- использует взвешенный ZNCC, градиенты и локальную подгонку яркости;
- выполняет быстрый prefilter кандидатов;
- уточняет смещение до субпиксельного уровня;
- проверяет forward-backward consistency;
- отбрасывает пространственно несогласованные векторы.

Результат:
- набор надежных контрольных точек;
- их смещения;
- confidence для каждой точки;
- диагностика принятых и отклоненных точек.

### `dsa_v2/motion.py`
Выбор модели движения.

Что делает:
- фильтрует выбросы;
- оценивает `rigid`, `affine` и `nonrigid` поведение;
- использует RANSAC для устойчивости;
- считает остатки, coherence, confidence и вариативность смещений;
- в рабочем режиме предпочитает `nonrigid`, если данных достаточно.

Результат:
- объект `MotionDecision`, где указана выбранная модель движения.

### `dsa_v2/field.py`
Построение и применение поля деформации.

Что делает:
- строит поле смещений для `rigid`, `affine` или `nonrigid`;
- для `nonrigid` использует интерполяцию по контрольным точкам;
- сглаживает поле;
- применяет confidence gate;
- защищает сосудистую область от ложных локальных смещений;
- деформирует изображение `mask`.

### `dsa_v2/subtraction.py`
Субтракция, коррекция и метрики.

Что делает:
- считает `live - warped_mask`;
- выполняет локальную коррекцию яркости warped-mask по надежному фону;
- подавляет низкочастотную фоновую дымку после субтракции;
- считает численные метрики качества.

Основные метрики:
- улучшение фона;
- сохранение сосудистого сигнала;
- контраст сосудов к фону;
- среднее и максимальное смещение.

### `dsa_v2/diagnostics.py`
Диагностика контрольных точек.

Что делает:
- анализирует, в каких областях находятся точки;
- добавляет к match-диагностике информацию о ROI, сосудах, safe-mask и весах;
- формирует сводку по принятым и отклоненным точкам.

### `dsa_v2/pipeline.py`
Главный файл алгоритма.

Что делает:
- связывает все этапы в один процесс;
- строит маски;
- выполняет предварительную глобальную регистрацию;
- выбирает контрольные точки;
- запускает matching;
- выбирает модель движения;
- строит поле деформации;
- деформирует mask;
- выполняет субтракцию и коррекции;
- считает метрики;
- сохраняет компактный набор результатов.

Сохраняемые файлы:
- `dsa_no_compensation.png`;
- `dsa_compensated.png`;
- `dsa_vessel_enhanced.png`;
- `vessel_support_overlay.png`;
- `displacement_magnitude.png`;
- `match_vectors.png`;
- `motion_report.json`;
- `summary.csv`.

### `dsa_v2/__init__.py`
Минимальный экспорт пакета.

Что делает:
- открывает наружу `PipelineConfig`;
- открывает `default_config_for_shape`;
- открывает `run_pipeline`.

## Основной поток данных

```text
mask/live
-> ROI и сосудистые маски
-> контрольные точки на фоне
-> local matching
-> выбор non-rigid/rigid/affine модели
-> поле смещений
-> warp(mask)
-> live - warped_mask
-> intensity correction
-> haze correction
-> метрики и сохранение результатов
```
